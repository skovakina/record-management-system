Record Management System

Travel-agent desktop app (Python 3 + Tkinter). Two views: the module architecture, and how a record flows through the system.

1 · Architecture

Layered design — the GUI never touches JSON directly; it goes through RecordStore, which owns the domain records and delegates file I/O to storage.

User Interface (Tkinter) Logic Storage (JSON)

2 · Data flow

A record's round trip on startup (load) and on a create / edit / delete action. Every write persists immediately and reloads from disk, so memory and JSON stay in sync.
